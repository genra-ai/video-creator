# 先选角，再拍剧

两阶段短剧制作流程。第一阶段：在创建任何剧情镜头之前，先为每个角色生成一张正面参考图，锁定外观。第二阶段：以这些参考图作为 I2I 锚点，贯穿整个剧情制作。

## 核心原则

- **第一阶段必须完成并经用户确认后，方可进入第二阶段** — 若参考图与角色设定不符，须重新生成
- **画风优先级最高**：必须写在项目消息的第一行。默认：`Art style: realistic cinematic`
- **不得改写用户剧本** — 只补充视觉信息
- **第一阶段只生成图片，绝对不为参考镜头生成视频**
- **每个角色只有一个镜头（仅正面），每个镜头只有首帧，无尾帧**
- **剧情镜头只用首帧** — 第二阶段设置完成后清除所有尾帧
- **按角色分组检查图片**，不按镜头顺序逐一检查
- **人种必须与原始素材一致** — 仔细阅读角色文件；"欧美人物形象"指西方/白人外貌，不是东亚外貌

---

## 第一阶段：角色参考镜头

### 步骤一：准备镜头列表

如果用户提供了角色描述文件，**不要原文照发**。从中提取每个角色与外貌/声音相关的信息，简化后发送到平台。保留内容：身份/种族（吸血鬼、妖精等，直接影响外貌气质）、容貌特征、发型发色、肤色、体型、服装、标志性配饰、声音/气质描述。删除内容：性格细节、背景故事、人物关系、剧情作用等与视觉无关的信息。

在简化后的角色描述之后，附上镜头列表，只需注明：
- 镜头编号
- 角色名称

示例：
```
镜头1：SELENE STERLING 正面参考，时长1秒
镜头2：WILLIAM DE VILLE 正面参考，时长1秒
镜头3：MINA MURDOCK 正面参考，时长1秒
...（每个角色1个镜头，共N个镜头）
```

不要添加任何额外的外观描述、服装细节或打光说明——平台会从上方角色描述中自行提取。

### 步骤二：创建参考项目并发送角色镜头

创建一个名为 `[原项目名] — Cast` 的新项目。发送一条消息，包含：
- 第一行写画风
- **每个角色一个镜头**——仅正面（N个角色 = N个镜头）
- 每个镜头**只有首帧，无尾帧**
- 镜头时长：**每个1秒**
- **不包含视频描述** — 第一阶段只生成图片，参考镜头绝不生成视频

将视觉要求作为**全局约束**写在镜头列表之前，不要在每个镜头中重复：

```
【全局约束】所有参考镜头首帧描述：人物全身肖像，正面站立，面向镜头，
表情自然放松，双臂自然垂放。纯白背景，无任何环境元素和道具，
均匀正面打光无阴影，全身从头到脚完整入画，人物居中。

镜头1：[角色名] 正面参考，时长1秒
镜头2：[角色名] 正面参考，时长1秒
镜头3：[角色名] 正面参考，时长1秒
...（每个角色1个镜头，共N个镜头）
```

> 不要在单个镜头描述中添加任何外观细节（发色、服装等）——平台会从上方角色文件原文中提取。

### 步骤三：核验参考镜头

图片生成后，下载并逐一检查每个角色的正面参考帧：

| 检查项 | 要求 |
|--------|------|
| 服装颜色 | 与设定完全一致——偏差则重新生成 |
| 发型 | 长度、质感、颜色正确 |
| 标志性细节 | 可见且正确（眼镜、首饰等） |
| 全身可见 | 从头到脚完整入画 |
| 背景 | 干净白色，无道具或场景元素 |
| 朝向 | 完全正面，未侧身或转身 |

不断重新生成（添加更明确的描述）直到每个角色外观锁定。

**第一阶段参考镜头不得生成视频。所有参考图确认通过前，不得进入第二阶段。**

---

## 第二阶段：剧情制作

### 步骤四：分段发送原始剧本

**不得创建新项目。** 在同一个 Cast 项目中继续。**不自行编写分镜描述**——直接发送原始剧本段落，让平台自动解析分镜。

**分段规则**：
- 每条消息发送约对应 **10个分镜** 的剧本内容（大约10–12行动作/台词）
- 按场景或自然段落断开，**不在场景中间截断**
- 第一段附完整角色参考块；后续段附简短角色外观提醒

**语言规则**：
- **场景描述、动作、旁白**：统一用**中文**发送
- **角色对话台词**：保留台词原文语言（中文台词用中文，英文台词用英文）
- 发送前将剧本中的英文场景描述/舞台指示翻译为中文，台词部分不翻译

**每段消息结构**：
```
Art style: [与第一阶段完全一致]

本集出场角色参考（请在生成图片时保持以下角色外观与对应参考镜头一致）：
- [角色名] → 参考本项目第[N]号镜头首帧
- [角色名] → 参考本项目第[M]号镜头首帧
...（仅列出本段中出现的角色）

---

[本段剧本——场景描述/动作/旁白用中文，台词保留原文语言，一字不改]
```

> **为什么用同一项目**：角色参考帧（镜头1–N）已存在于本项目中。直接将其作为 I2I 锚点引用，是锁定外观最可靠的方式。创建新项目会丢失这个直接引用。

> **音频与视频生成**：只为剧情镜头（第 N+1 号起）生成音频和视频。参考镜头（第1–N号）只有图片，绝不生成音频或视频。

> **发送节奏**：每段发送后等待平台完成解析，再发下一段，避免镜头顺序混乱。

> **台词拆镜**：剧本中出现对话台词时，在发送的剧本段落末尾补充说明，要求平台将每句台词单独拆分为一个镜头，不要将多句台词合并在同一镜头中。

### 步骤五：过滤检查

发现问题立即修复。

- **台词**：核实每行台词归属的角色是否正确；旁白不得归属到具体角色
- **镜头**：确认景别解析正确；台词镜头时长须 ≥ 自然朗读时间 + 0.5秒；反应镜头最短2–3秒

### 步骤六：清除所有尾帧

发消息让平台清除剧情项目中的所有尾帧——只保留首帧。事后确认。若自动清除失败，在生成面板中手动取消勾选尾帧。

### 步骤七：按角色分组检查图片一致性

将每个角色的剧情帧与其**第一阶段参考帧**逐一对比。参考帧是唯一标准。

| 问题 | 常见偏差 | 修复方式 |
|------|----------|----------|
| 服装颜色 | 白色→米白，深蓝→黑色 | 以参考帧做 I2I；重新指定颜色 |
| 发型 | 长发变短，刘海消失 | 以参考帧做 I2I；重新强调 |
| 标志性细节 | 眼镜/首饰消失 | 以参考帧做 I2I；明确强调 |
| 年龄感 | 角色显老/显年轻 | 重新生成；添加年龄锚定词 |
| 玻璃眼 | 近景/大特写镜头 | I2I："眼神清晰锐利，有神" |
| 手部变形 | 握持/指向镜头 | 重新生成；简化手部描述 |
| 对话方向相同 | 正反打镜头 | 明确指定每个角色的朝向 |

同一场景的连续镜头：同时核实景深一致性。每个镜头最多重试一次；仍不理想则接受并继续。

### 步骤八：生成音频

**声音描述公式**：`[性别+年龄] + [音色质感] + [情感基调] + [语速]`

| 角色类型 | 示例 |
|----------|------|
| 年轻女主（压抑情感） | 年轻女性，明亮音色，克制压抑，语速缓慢，停顿感强 |
| 中年男性（疲惫/愧疚） | 成熟男性，低沉略沙哑，疲惫质感，语速缓慢，尾音拖长 |
| 年轻人（放松） | 年轻男性，明亮音色，随意自然，语速适中 |
| 强势/反派 | 成熟男性，低沉有力，语速较快，咬字坚定 |

语速跟随场景情绪——**不得**默认使用"快速"。声音克隆：上传5–10秒参考音频；将返回的 asset_id 填入该角色的 voice 字段。

### 步骤九：生成视频

所有音频完成后，生成所有剧情视频。

### 步骤十：预览与导出

- 场景切换：剪切干净，无混合瑕疵
- 多角色对话：口型同步与对应角色一致
- 景别与剧本意图匹配（全景/中景/近景）
- 若尾帧仍导致画面异常：再次清除，仅重新生成受影响的镜头

所有镜头通过后导出。
